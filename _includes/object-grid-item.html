Shared by both grids in objects-index.html (browse-and-search and plain) so the two cannot drift apart. Jekyll include scope does not reliably inherit the parent layout's `lang` assign (see _includes/story-step.html for the same note), so both `lang` and `object` are passed in explicitly rather than relied on from the including template's scope. Parameters: object - the site.objects entry to render lang - the active language pack (site.data.languages[...]) show_filters - when true, stamps the data-* filter attributes (data-media-type, data-medium, data-creator, data-period, data-subjects, data-year, data-title, data-description) that objects-filter.js reads; omit/false for the plain grid Video/Audio objects render a static icon (see the media_type branches below) instead of the thumbnail cascade, and the outer `.collection-item-image` wrapper withholds `data-iiif-manifest` for them: `iiif_manifest` is only a `source_url` alias kept for backward compat (scripts/telar/processors/objects.py), and for a video object it holds the YouTube/Vimeo/Drive page URL, not IIIF JSON. Letting that attribute through would make the inline script in objects-index.html fetch the page URL as if it were a manifest, failing (CORS/parse error) on every load — hence the media_type guard on the div.
