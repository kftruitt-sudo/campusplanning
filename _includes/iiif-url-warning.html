{%- comment -%} IIIF URL mismatch warning — a self-diagnosing banner that catches the most common reason locally hosted images fail to appear: the address the site is being viewed at no longer matches the base URL baked into its IIIF manifests when the tiles were generated. The markup here is the hidden alert box; the diagnostic logic itself lives in assets/js/iiif-url-warning.js, a standalone classic script loaded below that decides whether to reveal it. This include stays self-contained: it carries the Liquid-dependent values the script needs (the objects.json and IIIF manifest base paths, plus the localized singular/plural affected-image strings) onto window.telarIIIFWarningConfig in a short inline script, then loads the external module. See the module's own header for how it uses that configuration to detect and report the mismatch.
