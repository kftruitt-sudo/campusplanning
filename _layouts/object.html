--- layout: default --- {%- comment -%} Object layout — the detail page for a single collection object, rendered inside _layouts/default.html (the base page frame). One template serves all three media types (Image, Video, Audio); the media type, injected into frontmatter by generate_collections.py as `page.media_type`, switches which viewer and which author-facing tools appear. This is the large (~1,230-line) object page, long mostly because each media branch carries its own viewer wiring, metadata block, and copy-to-clipboard scripts; the shared algorithmic pieces (theme-colour maths, video embed resolution, coordinate normalisation) live in assets/js/object-theme.js, assets/js/video-embed.js, and assets/js/telar-story/iiif-viewer.js. What it renders — a two-column layout: a viewer on the left and a metadata sidebar on the right. The viewer branches by media type — an OpenSeadragon IIIF viewer for images (with a collapsible "identify coordinates" panel that reports live x/y/zoom/page values for authors building stories), an embedded YouTube/Vimeo/Google Drive iframe for video (with a clip-time picker), or a vendored WaveSurfer waveform for audio (also with a clip picker). The sidebar shows title, description, and a definition list of metadata (creator, period, medium, dimensions, location, credit, plus any extra CSV columns surfaced via `page.extra_metadata`), the IIIF manifest or provider/source/embed URLs, the object's long-form `{{ content }}`, and a list of stories the object appears in. Key context — `page.media_type` (defaults to Image) drives every branch; `page.object_id` locates locally generated IIIF tiles and audio files; `page.source_url`/`page.iiif_manifest` point at remote media; `lang` supplies all user-facing strings; `site.objects` is scanned to find related stories. Author tooling strings live under `lang.object.coordinates` and `lang.object.clip_picker`. Non-obvious behaviour — object pages deliberately skip the full telar-story.js bundle (which is story-only) and instead import the IIIF viewer wrapper directly from its ESM source, and they re-enable OpenSeadragon's native scroll/click-to-zoom because there is no Lenis scroll engine here to conflict with. URLs interpolated into inline scripts are jsonified and guarded against `` breakout, and only http(s) source URLs render as live links so a javascript:/data: URL degrades to plain text.
